## 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-CONTACT |
| 画面名 | お問い合わせ |
| ファイル | contact.html |
| URL | /contact.html |
| 認証 | 要ログイン |

## 画面概要

ユーザーがサービスに関する問い合わせを送信できるフォーム画面。カテゴリ・関連サービス・件名・本文を入力して送信すると、受付番号が発行され送信完了画面に切り替わる。

## 表示項目

### お問い合わせフォーム（#contactFormView）

| No. | 項目 | 種別 | 必須 | 説明 |
|-----|------|------|------|------|
| 1 | カテゴリ | ドロップダウン | ○ | 問い合わせの分類を選択 |
| 2 | 関連サービス | ドロップダウン | — | 契約中サービスから選択（任意） |
| 3 | 件名 | テキスト入力 | ○ | 問い合わせの件名 |
| 4 | お問い合わせ内容 | テキストエリア（6行） | ○ | 問い合わせの詳細 |
| 5 | 送信ボタン | ボタン | — | 「送信する」（プライマリ） |

#### カテゴリ選択肢

| 値 | 表示テキスト |
|-----|-------------|
| （空） | 選択してください |
| billing | 料金・請求について |
| technical | 技術的なお問い合わせ |
| contract | 契約・解約について |
| other | その他 |

#### 関連サービス選択肢

| 値 | 表示テキスト |
|-----|-------------|
| （空） | なし |
| （動的） | ステータスが「active」の契約中サービス名を `initContact()` で動的追加 |

### 送信完了画面（#contactCompleteView）

初期状態は非表示。送信成功後に表示される。

| No. | 項目 | 説明 |
|-----|------|------|
| 1 | 完了アイコン | 緑色チェックマーク（✔） |
| 2 | 見出し | 「お問い合わせを受け付けました」 |
| 3 | 受付番号 | 「TK-」+ タイムスタンプ下8桁で自動生成 |
| 4 | 案内文 | 「担当者より2営業日以内にご連絡いたします。」 |
| 5 | 再送信リンク | 「新しいお問い合わせ」ボタン（contact.htmlへ遷移） |

## 操作仕様

### フォーム送信

`submitContact(event)` により以下の処理を実行する。

| ステップ | 処理 |
|----------|------|
| 1 | デフォルトのフォーム送信を抑止（`preventDefault`） |
| 2 | 件名・本文の空チェック。空の場合は該当フィールドに `has-error` クラスを付与しエラーメッセージを表示 |
| 3 | バリデーション通過後、受付番号（TK-XXXXXXXX）を生成 |
| 4 | フォーム表示を非表示にし、送信完了画面を表示 |

## バリデーション

| 対象フィールド | ルール | エラーメッセージ |
|---------------|--------|-----------------|
| カテゴリ | 未選択不可（HTML required） | ブラウザ標準メッセージ |
| 件名 | 空文字不可 | 「件名を入力してください」 |
| お問い合わせ内容 | 空文字不可 | 「お問い合わせ内容を入力してください」 |

## 画面遷移
